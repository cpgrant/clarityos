# ClarityClaw v1.3 Checklist

Status: complete

Goal:
Make the browser-first assistant materially more useful by improving grounding, answer quality, and tool-guided task structure before expanding surface area.

## Slice 1: Assistant Grounding And Structured Answers

Status: complete

Focus:

- make repo and docs questions use explicit retrieval paths first
- improve answer structure for project, architecture, and operator questions
- reduce generic and repetitive answers in the assistant surface

Candidate work:

- add explicit repo-context injection for selected docs and files
- improve assistant prompting for concise, grounded, structured responses
- prefer repo search and file-range tools for codebase questions
- add assistant response modes such as brief, structured, and deep dive if needed

Acceptance cues:

- repo questions cite relevant files instead of producing generic software answers
- assistant answers are clearer, less repetitive, and easier to scan
- browser assistant behavior better matches operator expectations

## Slice 2: Tool-Guided Assistant Quality

Status: complete

Focus:

- make the assistant use the most valuable explicit tools more reliably
- improve retrieval and inspection capability for real tasks
- add controlled external retrieval and runtime-ID inspection without creating a second execution path

Candidate work:

- add controlled external retrieval such as `fetch_url` or `http_get`
- improve runtime-inspection tool use for troubleshooting and operator questions
- tighten tool result shaping for more readable assistant outputs

Current progress:

- assistant grounding now uses guarded repo excerpts from the safe file tools
- runtime questions with explicit workflow or session IDs can add compact inspection summaries
- a controlled `fetch_url` tool now exists for the `researcher` agent with explicit domain allowlisting
- grounding helper tool use now checks the active agent's tool allowlist and policy before adding context
- final prompt context now includes a compact working-answer frame to make tool-derived answers easier to structure
- comparison and planning questions now get an explicit suggested answer structure instead of relying on generic formatting

Acceptance cues:

- the assistant can answer repo, runtime, and bounded research questions more reliably
- tool outputs are easier to turn into good final answers

## Slice 3: Browser Surface Quality And Validation

Status: complete

Focus:

- improve the experience and trustworthiness of `/assistant`, `/operator`, and `/widget`
- validate the assistant behavior in real browser flows

Candidate work:

- improve browser-side clarity around agent choice, session state, and errors
- add end-to-end browser smoke tests for assistant, operator, and widget flows
- tighten output readability and copy/paste ergonomics where needed

Current progress:

- a browser-surface smoke suite now exercises `/assistant`, `/operator`, and `/widget`
- the smoke suite uses real session, workflow, control-plane, and widget route flows with temp persisted state
- regressions now have a browser-facing integration layer beyond the route passthrough tests

Acceptance cues:

- the three browser surfaces are easier to test and trust
- regressions are caught beyond the API-only test layer

## Slice 4: Release Path For Assistant Quality

Status: complete

Focus:

- define what it means for `v1.3` to be complete and measurably better

Candidate work:

- add explicit release criteria for grounding and answer quality
- document the narrow supported use cases improved by `v1.3`
- define what remains intentionally out of scope

Current progress:

- the `v1.3` release path is now defined in `docs/v1.3-release-path.md`
- release gates now cover validation, grounding posture, final-answer structure, browser-surface trust, and supported use-case boundaries
- the milestone now has a concrete definition of what “measurably better than v1.2” means

Acceptance cues:

- `v1.3` can be described as a quality milestone, not just another surface milestone
- release status can be justified by concrete assistant-quality improvements
