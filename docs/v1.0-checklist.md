# ClarityOS v1.0 Checklist

Concrete implementation checklist for `v1.0` release readiness and the first production profile.

## Slice 1: Soak, Load, And Recovery Validation

- [ ] `tests/`
  Add repeatable soak and long-running recovery coverage for queue, worker, workflow, and incident paths.
- [ ] `tests/`
  Add restart and reclaim drills that exercise expired leases, dead-letter recovery, retry wait, and operator replay at higher concurrency.
- [ ] `scripts/`
  Add lightweight validation runners for pre-release recovery and failure-drill execution.
- [ ] `README.md`
  Document how to run the non-unit validation suite locally before release.

## Slice 2: Rollout Defaults And Deployment Profile

- [ ] `config/agents.yaml`
  Define a production-oriented default agent posture with conservative budgets and bounded capabilities.
- [ ] `config/policies.yaml`
  Tighten the shipped production policy profile and document what remains opt-in.
- [ ] `README.md`
  Add a production profile section covering env vars, TLS/front-proxy assumptions, persisted directories, and rollout defaults.
- [ ] `requirements.txt`
  Add any deployment/runtime dependencies only if the production profile chooses them explicitly.

## Slice 3: Operator Governance And Maintenance Playbooks

- [ ] `docs/`
  Add operator playbooks for migration, queue cleanup, worker repair, workflow recovery, and incident response.
- [ ] `api/main.py`
  Add any additional operator-oriented status or maintenance endpoints only if the playbooks need them.
- [ ] `README.md`
  Link the operator playbooks from the main documentation.

## Slice 4: First Production Path And Release Criteria

- [ ] `docs/roadmap.md`
  Define the narrow first production use case and the explicit “done means shipped” release gates.
- [ ] `README.md`
  Summarize the intended first production profile and what is deliberately still out of scope.
- [ ] `docs/history/`
  Archive the `v0.9` snapshot once `v1.0` scope settles and keep the milestone trail aligned.
