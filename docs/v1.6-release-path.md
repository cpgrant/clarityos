# v1.6 Release Path

This document defines the narrow multi-agent-work-quality release path for ClarityClaw `v1.6` and the release gates required to call that path measurably better than `v1.5`.

## First Multi-Agent-Quality Use Case

ClarityClaw `v1.6` is aimed at a narrow but stronger bounded multi-agent shape:

- single-tenant
- self-hosted
- browser-first and API-first
- workflow-backed
- operator-visible
- delegation-bounded
- synthesis-aware
- auditable rather than opaque

In concrete terms, the first supported `v1.6` multi-agent-quality path is:

`A self-hosted assistant and operator runtime where parent workflows can delegate bounded child tasks through explicit contracts, inspect compact child-result synthesis, and audit delegated failures and contract gaps without turning into open-ended autonomous agent swarms.`

## Supported Shape

The `v1.6` release path supports:

- the web-first assistant at `/assistant`
- the embeddable widget at `/widget` and `/widget.js`
- the operator console at `/operator`
- bounded child workflow delegation through:
  - explicit child contracts with task intent, expected output, and completion criteria
  - capability and tool bounds inherited from the parent workflow policy
  - early validation for invalid or empty delegated task briefs
- bounded supervisor-style synthesis through:
  - compact child-result rollups based on persisted workflow state
  - next-action guidance such as `wait_for_children`, `review_partial_results`, `review_failed_children`, and `synthesize_results`
  - successful-child and active-child summaries built from real child memories, artifacts, and current step state
- explicit delegated-run auditability through:
  - per-child audit summaries for ownership, contract completeness, and output gaps
  - delegation-denied trace visibility in incident review
  - contained-vs-propagated failure counts for delegated children

## Deliberately Out Of Scope

The `v1.6` release path does not claim support for:

- open-ended autonomous agent swarms
- unbounded delegation depth or fan-out
- hidden supervisor reasoning that exists only in prompts
- broad proactive background execution
- many-agent orchestration without operator-visible boundaries
- hosted multi-tenant agent infrastructure

`v1.6` is the bounded multi-agent-quality release, not the autonomous swarm release.

## Release Gates

ClarityClaw `v1.6` is ready for this bounded multi-agent-quality path only if all of the following are true.

### Gate 1: Validation Passes

- `scripts/run_release_validation.sh --full` passes cleanly.
- `python -m unittest discover -s tests -v` passes cleanly.
- `python -m unittest tests.test_workflow tests.test_workflow_runner tests.test_api tests.test_control_plane -v` passes cleanly.

### Gate 2: Delegation Contracts Stay Explicit

- child workflows persist task intent, expected output, and completion criteria in workflow state
- child delegation still respects explicit capability and tool bounds
- invalid or empty delegated task briefs fail before child execution begins

### Gate 3: Child Result Synthesis Stays Bounded

- child synthesis is derived from persisted child workflow state rather than hidden prompt-only summaries
- successful, active, failed, and missing child states are distinguishable in workflow inspection
- synthesis guidance remains compact and operator-readable rather than sprawling into transcript replay

### Gate 4: Auditability Is Credible

- operators can identify child ownership, contract gaps, result gaps, and delegated failure state from control views
- delegation-denied traces appear in incident review rather than disappearing into generic runtime errors
- contained child failures can be distinguished from failures that propagated to the parent workflow

### Gate 5: The Supported Use Case Is Believable

Operators can credibly use `v1.6` for:

- planner or supervisor workflows that hand bounded research or summarization tasks to child agents
- reviewing which child results are ready before composing a parent answer
- identifying whether a child failed because of delegation bounds, missing output, or normal runtime failure
- continuing bounded multi-agent runs without losing inspectability

### Gate 6: Scope Remains Narrow

The release must stay within the declared `v1.6` support boundary:

- better bounded delegation, not open-ended autonomy
- explicit contracts and audits, not hidden multi-agent magic
- compact synthesis and operator visibility, not opaque swarm coordination
- supportable planner or supervisor workflows, not broad autonomous agent ecosystems

If a candidate depends on open-ended swarms or hidden orchestration logic, it belongs after `v1.6`.

## Done Means Better Delegation, Not More Autonomy

For `v1.6`, “done” does not mean ClarityClaw becomes a broad autonomous multi-agent platform.

It means:

- child delegation is clearer and easier to validate
- parent workflows can inspect bounded child-result synthesis before deciding what to do next
- operators can audit delegation gaps, failures, and denied runs credibly
- one believable bounded multi-agent workflow path is measurably better than `v1.5`

That is the line between “supportable multi-agent work quality” and “open-ended autonomous orchestration.”
