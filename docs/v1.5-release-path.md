# v1.5 Release Path

This document defines the narrow memory-and-continuity release path for ClarityClaw `v1.5` and the release gates required to call that path measurably better than `v1.4`.

## First Memory-Maturity Use Case

ClarityClaw `v1.5` is aimed at a narrow but stronger long-running assistant shape:

- single-tenant
- self-hosted
- browser-first
- session-backed
- operator-visible
- continuity-aware
- bounded rather than opaque

In concrete terms, the first supported `v1.5` memory-maturity path is:

`A self-hosted assistant and operator runtime where longer-running browser sessions can be compacted intentionally, carried forward through explicit continuity summaries, and inspected through visible continuity budgets rather than hidden prompt stuffing.`

## Supported Shape

The `v1.5` release path supports:

- the web-first assistant at `/assistant`
- the embeddable widget at `/widget` and `/widget.js`
- the operator console at `/operator`
- explicit continuity compaction through:
  - operator-invoked session continuity compaction
  - persisted compaction metadata with source message, memory, and workflow references
  - bounded retained-message windows
- explicit session continuity summaries through:
  - persisted session-level continuity summary state
  - bounded carry-forward of recent messages, scoped memories, and related workflow identifiers
  - continuity prompt context injected for later session turns
- explicit continuity budgeting through:
  - compact-now and recompact-now recommendations
  - carry-forward window counts and summary-size visibility
  - operator-visible action hints through the session control view
- operator maintenance guidance for:
  - continuity inspection before archive/prune decisions
  - continuity compaction for long-running sessions
  - continuity-aware session cleanup

## Deliberately Out Of Scope

The `v1.5` release path does not claim support for:

- hidden long-context dumping into model prompts
- automatic preference learning from weak behavioral signals
- proactive autonomous task execution based on inferred user intent
- broad multi-user shared-memory collaboration
- hosted multi-tenant continuity infrastructure
- consumer-grade background assistant behavior

`v1.5` is the memory-maturity release, not the autonomous personal-agent release.

## Release Gates

ClarityClaw `v1.5` is ready for this memory-maturity path only if all of the following are true.

### Gate 1: Validation Passes

- `scripts/run_release_validation.sh` passes cleanly.
- `python -m unittest discover -s tests -v` passes cleanly.
- `python -m unittest tests.test_session tests.test_control_plane tests.test_api -v` passes cleanly.

### Gate 2: Compaction Stays Explicit

- continuity compaction is invoked intentionally rather than happening as hidden prompt mutation
- compaction state is persisted in session metadata rather than inferred only at runtime
- compacted message ids, retained message ids, and related source references stay visible

### Gate 3: Carry-Forward Stays Bounded

- continuity summaries carry only bounded recent-message, memory, and workflow context
- continuity prompt context is derived from persisted state rather than opaque transcript trimming
- continuity summaries are inspectable in the control plane and operator surface

### Gate 4: Budgeting Is Credible

- sessions can surface whether they need first compaction, recompaction, summary refresh, or no action
- operator-visible counts explain why a recommendation exists
- compact/recompact decisions are based on explicit thresholds, not hidden heuristics

### Gate 5: Long-Running Session Support Is Believable

Operators can credibly use `v1.5` for:

- browser-first assistant sessions that stay coherent over longer threads
- reviewing explicit continuity summaries before continuing a thread
- deciding when to compact or recompact based on visible budget pressure
- inspecting what continuity context is being carried forward and why

### Gate 6: Scope Remains Narrow

The release must stay within the declared `v1.5` support boundary:

- better continuity, not hidden autonomy
- explicit summaries, not opaque memory magic
- operator-visible budgeting, not silent background behavior
- supportable long-running browser sessions, not broad autonomous assistant sprawl

If a candidate depends on implicit preference learning or proactive background task execution, it belongs after `v1.5`.

## Done Means More Coherent, Not More Autonomous

For `v1.5`, “done” does not mean ClarityClaw becomes a broad personal-agent system.

It means:

- longer-running sessions can be compacted intentionally instead of growing without structure
- continuity summaries are explicit, inspectable, and reused for bounded carry-forward
- operators can see when continuity is healthy, near budget, or due for action
- one believable long-running browser-session path is measurably better than `v1.4`

That is the line between “session-backed assistant with memory maturity” and “open-ended autonomous assistant behavior.”
