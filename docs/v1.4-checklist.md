# ClarityOS v1.4 Checklist

Status: planned

Goal:
Strengthen the assistant's explicit tool layer so it can inspect, retrieve, and act more usefully without losing ClarityOS's inspectability and policy boundaries.

## Slice 1: Tool Registry Structure

Status: planned

Focus:

- make the growing tool layer easier to extend and reason about
- reduce monolithic tool-file pressure without introducing hidden behavior

Candidate work:

- split tool handlers by domain such as repo, runtime, memory, and web
- keep a single explicit registry surface
- tighten tool metadata for capability, path, and domain handling

Acceptance cues:

- the tool layer is easier to extend without becoming messy
- policy and agent allowlisting remain explicit and visible

## Slice 2: Retrieval Capability Depth

Status: planned

Focus:

- improve the quality and usefulness of safe retrieval tools
- support more real assistant research and operator tasks

Candidate work:

- deepen repo and runtime inspection tools
- refine `fetch_url` and related bounded retrieval behavior
- improve tool output shapes for downstream answer quality

Acceptance cues:

- the assistant can inspect repo and runtime state more usefully
- retrieval outputs are easier to transform into grounded answers

## Slice 3: Safer Action Tools

Status: planned

Focus:

- add narrow, supportable action capability without broad unsafe execution

Candidate work:

- evaluate scoped write or patch tools
- consider narrow operator-safe action helpers
- avoid broad exec-by-default behavior

Acceptance cues:

- ClarityOS becomes more useful for real tasks without losing explicit safety boundaries

## Slice 4: Release Path For Tool Maturity

Status: planned

Focus:

- define what tool-maturity improvement means for `v1.4`

Candidate work:

- document the narrow supported use cases improved by `v1.4`
- define validation and release gates for the stronger tool layer
- define what remains deliberately out of scope

Acceptance cues:

- `v1.4` can be described as tool-maturity progress rather than just more tools
