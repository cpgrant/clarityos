# ClarityClaw v1.5 Snapshot

Milestone snapshot for `v1.5`, the memory-and-continuity maturity release.

## Status

- Release: `v1.5`
- Focus after this release: `v1.6` planning
- Next target after this release: multi-agent work quality

## What v1.5 Added

- explicit session continuity compaction helpers instead of silent prompt-only transcript trimming
- persisted compaction metadata with source message, memory, and workflow references
- session-level continuity summaries that can be reused for later assistant turns
- bounded carry-forward rules for recent messages, continuity memories, and related workflow context
- continuity budgeting and operator-facing recommendations such as `compact_now`, `recompact_now`, and `within_budget`
- an explicit `v1.5` release path for the first narrow long-running assistant use case

## Acceptance Criteria Met

- Long-running assistant sessions can be compacted intentionally rather than growing without structure.
- Continuity summaries are explicit, inspectable, and carried forward in bounded form.
- Operators can see continuity budgets, counts, and recommended next actions instead of inferring continuity state indirectly.
- The supported `v1.5` use case stays narrow and credible: better continuity for browser-first sessions, not hidden autonomy.
- One believable memory-maturity path is documented and measurably better than the `v1.4` baseline.

## Supported Shape

`v1.5` supports a narrow but stronger memory-and-continuity path:

- browser-first assistant at `/assistant`
- optional embeddable widget through `/widget` and `/widget.js`
- operator console through `/operator`
- explicit continuity compaction and continuity-summary state for longer-running sessions
- bounded carry-forward for later assistant turns
- operator-visible continuity budgeting and compact/recompact guidance

It deliberately does not yet include:

- hidden long-context dumping into prompts
- automatic preference learning from weak behavioral signals
- proactive autonomous task execution based on inferred user intent
- broad multi-user shared-memory collaboration
- hosted multi-tenant continuity infrastructure
