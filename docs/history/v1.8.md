# ClarityClaw v1.8 Snapshot

Milestone snapshot for `v1.8`, the careful external-integration release for narrow email triage.

## Status

- Release: `v1.8`
- Focus after this release: `v1.9` planning
- Next target after this release: product refinement and supported workflows

## What v1.8 Added

- a bounded `/intake/email` adapter that hands external email input to the existing session and workflow runtime
- explicit email-origin metadata across intake, session, workflow, artifact, and operator state
- structured `email_triage` artifacts with bottom line, urgency, suggested bucket, next action, and optional draft reply
- operator-visible draft-review approvals for email triage instead of hidden outbound behavior
- explicit `email_draft_handoff` artifacts that mark approved drafts as ready for manual follow-through
- an explicit `v1.8` release path for the first narrow external-integration use case

## Acceptance Criteria Met

- One careful external integration exists without turning ClarityClaw into a broad many-channel product.
- Email triage results are structured, inspectable, and persisted as normal runtime state instead of existing only in prompt output.
- Operators can review, approve, deny, abort, and hand off draft suggestions through explicit runtime controls.
- The supported `v1.8` use case stays narrow and believable: bounded email triage with manual handoff, not autonomous mailbox management.
- One supportable external-integration path is documented and measurably better than the `v1.7` baseline.

## Supported Shape

`v1.8` supports a narrow but stronger external-integration path:

- browser-first assistant at `/assistant`
- optional embeddable widget through `/widget` and `/widget.js`
- operator console through `/operator`
- operator-authenticated email intake through `/intake/email`
- explicit `email_triage` artifacts tied to the normal workflow and session runtime
- explicit draft-review approvals through `/artifacts/{artifact_id}/email-draft-approval`
- explicit approved-draft handoff artifacts through `/artifacts/{artifact_id}/email-approved-draft-handoff`

It deliberately does not yet include:

- autonomous email sending
- mailbox mutation such as archive, label, delete, move, or reply-send
- full inbox synchronization or background mailbox mirroring
- broad many-channel expansion all at once
- hidden preference learning or silent inbox automation
