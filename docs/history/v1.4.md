# ClarityOS v1.4 Snapshot

Milestone snapshot for `v1.4`, the tool-maturity release.

## Status

- Release: `v1.4`
- Focus after this release: `v1.5` planning
- Next target after this release: memory and continuity maturity

## What v1.4 Added

- an explicit domain-split tool layer while preserving `runtime.tools` as the single registry facade
- stronger bounded retrieval outputs for repo, runtime, and controlled external sources
- a bounded `list_directory` helper and clearer retrieval metadata such as previews, counts, summaries, and matched-file context
- narrow runtime-maintenance action tools behind a dedicated `runtime_write` capability
- a dedicated `maintenance_operator` profile for explicit maintenance work instead of broad mutation power
- denial coverage and documentation that keep maintenance actions out of general assistant agents
- an explicit `v1.4` release path for the stronger tool layer and its scope boundaries

## Acceptance Criteria Met

- The tool layer is easier to extend without becoming monolithic or hiding registry behavior.
- Retrieval outputs are more useful for both assistant grounding and operator inspection than the `v1.3` baseline.
- Controlled external retrieval stays bounded by configured domains and explicit policy.
- The shipped action set stays narrow, operator-safe, and runtime-maintenance-only.
- One believable tool-maturity path is documented and measurably better before any broader mutation-platform expansion.

## Supported Shape

`v1.4` supports a narrow but more capable tool-maturity path:

- browser-first assistant at `/assistant`
- optional embeddable widget through `/widget` and `/widget.js`
- operator console through `/operator`
- stronger bounded retrieval for repo files, directory navigation, runtime inspection, and controlled external sources
- narrow maintenance actions through `maintenance_operator` for session, queue, worker, and workflow maintenance

It deliberately does not yet include:

- broad repo write helpers
- shell-like or exec-style mutation
- unrestricted maintenance powers for general assistant agents
- many-channel expansion
- hosted multi-tenant or marketplace behavior
