# ClarityOS v1.7 Snapshot

Milestone snapshot for `v1.7`, the packaged self-hosted deployment and operator-maturity release.

## Status

- Release: `v1.7`
- Focus after this release: `v1.8` planning
- Next target after this release: careful external integration without channel sprawl

## What v1.7 Added

- a first packaged startup baseline through `Containerfile`, `compose.yaml`, and `runtime.worker_loop`
- an explicit `CLARITYOS_STATE_ROOT` contract for packaged deployments
- operator-visible storage and backup posture, including per-directory preservation and regeneration guidance
- a runtime-posture dashboard summary that combines queue, worker, session, and storage signals into one bounded operator view
- an operator runtime playbook for repeated self-hosted operation
- an explicit `v1.7` release path for the first narrow packaged self-hosted deployment profile

## Acceptance Criteria Met

- A fresh environment can start the supported packaged profile through one documented path.
- Operators can explain what state must be preserved, what can be regenerated, and how packaged storage should be mounted.
- The operator console provides a faster and more credible runtime health summary than the `v1.6` baseline.
- The supported `v1.7` use case stays narrow and believable: repeatable self-hosting and operator maturity, not hosted-platform sprawl.
- One supportable packaged deployment path is documented and measurably better than `v1.6`.

## Supported Shape

`v1.7` supports a narrow but stronger self-hosted deployment path:

- browser-first assistant at `/assistant`
- optional embeddable widget through `/widget` and `/widget.js`
- operator console through `/operator`
- one documented packaged startup path for API and background worker execution
- one explicit state-root contract for packaged deployment, including `CLARITYOS_STATE_ROOT`
- operator-visible runtime posture, storage posture, and maintenance guidance for repeated self-hosted use

It deliberately does not yet include:

- hosted hub infrastructure
- broad SaaS-style multi-tenancy
- cluster-management breadth for every environment
- automatic cloud provisioning
- many-channel assistant expansion all at once
