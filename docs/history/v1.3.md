# ClarityOS v1.3 Snapshot

Milestone snapshot for `v1.3`, the assistant-quality release.

## Status

- Release: `v1.3`
- Focus after this release: `v1.4` planning
- Next target after this release: stronger tool maturity and capability depth

## What v1.3 Added

- assistant-surface repo grounding for browser-first project and roadmap questions
- guarded tool-guided grounding through safe repo tools, compact runtime inspection, and controlled external fetches
- stronger answer shaping for comparison, planning, and status-style questions
- browser-surface smoke validation for `/assistant`, `/operator`, and `/widget`
- clearer loading, empty, error, and owned-session state behavior across the browser surfaces
- an explicit quality-oriented release path with measurable assistant-quality gates

## Acceptance Criteria Met

- Browser-first assistant questions are more grounded and less generic than the `v1.2` baseline.
- Tool-guided grounding stays explicit and policy-aware rather than hidden or broad.
- Final answers are more structured and easier to scan for planning, comparison, and status questions.
- Browser surfaces are easier to trust because their loading, empty, error, and session states are clearer.
- One narrow browser-first assistant-quality path is documented and measurably better before broader assistant-platform expansion.

## Supported Shape

`v1.3` supports a narrow but more useful assistant shape:

- browser-first assistant at `/assistant`
- optional embeddable widget through `/widget` and `/widget.js`
- operator console through `/operator`
- grounded repo, roadmap, runtime, and bounded comparison questions
- guarded retrieval through safe repo tools, compact runtime inspection, and controlled fetch domains

It deliberately does not yet include:

- many-channel expansion
- broad arbitrary web research
- plugin marketplace behavior
- full autonomous coding or unrestricted execution
- hosted hub behavior
