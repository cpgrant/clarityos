# ClarityClaw v1.0 Snapshot

Milestone snapshot for `v1.0`, the trusted-runtime release.

## Status

- Release: `v1.0`
- Focus after this release: `v1.1` first assistant surface and session gateway
- Next target after this release: `v1.1` slice 1 session and conversation model

## What v1.0 Added

- repeatable release-validation drills for soak, recovery, replay, and safe-resume flows
- env-selectable production config files so deployment defaults do not overwrite local development configs
- production-oriented agent and policy example profiles
- operator profile introspection for environment, config, auth, and persisted-state posture
- production profile documentation and maintenance playbooks
- explicit first production path and measurable release gates

## Acceptance Criteria Met

- The runtime has an explicit production profile with rollout defaults, deployment guidance, and hardened operator posture.
- Recovery, retry, reclaim, and restart behavior are validated through repeatable load and failure drills beyond unit coverage.
- Operators have compact incident summaries, safe maintenance flows, and a documented operational playbook.
- Release criteria for the first real production use case are defined, measurable, and documented.

## First Production Path

`v1.0` supports a narrow, self-hosted, single-tenant, API-first workflow runtime for durable assistant and research tasks with operator-visible recovery and maintenance.

It deliberately does not yet include:

- end-user assistant UI
- multi-channel chat surfaces
- hosted multi-tenant operation
- plugin marketplace behavior
